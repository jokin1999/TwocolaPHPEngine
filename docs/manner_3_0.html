#Version3.0或以下版本

自3.1版本推出后，3.0及以下版本将不提供支持。主要原因有以下几点：

  • 代码不规范。
  • 路径解析引擎繁杂。
  • 模板变量区分大小写。
  • 框架级设置不够多，命名混乱。
  • 窝群系统不够完善，没有完整的个性化设置项。
  • 主要使用流程控制的开发模式，并非主流的oop模式。
  • 只有报错机制，没有报错系统，在程序出现错误时不能友好的显示。
  • 3.0设计的时候主要是出于学习，而3.1版本起主要出于运营，3.1+拥有更好的优化。
故3.0版本及以下不会继续提供支持，敬请谅解。
